# 빠른 시작

이 페이지의 목표는 모든 기능을 다 설명하는 것이 아닙니다. 가장 짧은 경로로 store hook 하나를 화면에 올리고, 올바른 업데이트 형태를 보여준 뒤, 다음에 무엇을 읽어야 하는지 연결하는 데 초점을 둡니다.

선호하는 패키지 매니저를 사용하여 패키지를 설치합니다:

```package-install
@ilokesto/state
```

## 1. store hook 만들기

가장 작은 `@ilokesto/state` 사용 예제는 plain initial state에서 시작합니다.

```tsx
import { create } from '@ilokesto/state';

const useCounter = create({ count: 0 });
```

## 2. slice를 읽고, 전체 상태를 쓴다

selector는 컴포넌트가 무엇을 읽을지를 정합니다. 하지만 반환되는 writer는 여전히 전체 store state를 갱신합니다.

```tsx
function Counter() {
  const [count, setState] = useCounter((state) => state.count);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setState((prev) => ({ ...prev, count: prev.count + 1 }))}>
        Increment
      </button>
    </div>
  );
}
```

이 지점이 처음엔 가장 헷갈리기 쉽습니다. selector가 `count`만 반환하더라도 writer는 이전 전체 상태를 받아 다음 전체 상태를 만들어야 합니다.

## 3. 쓰기가 로컬 이벤트를 넘어서면 컴포넌트 밖으로 뺀다

업데이트 로직이 버튼 핸들러를 넘어서 “액션”처럼 느껴지기 시작하면 `writeOnly()` 뒤로 빼는 편이 좋습니다.

```ts
const useCounter = create({ count: 0 });

export const increment = () => {
  const setState = useCounter.writeOnly();
  setState(state => ({ count: state.count + 1 }));
};
```

이렇게 하면 컴포넌트는 더 작아지고, store 동작은 재사용하거나 테스트하기 쉬워집니다.
