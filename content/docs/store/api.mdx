# Store API

The package exports one public class: `Store<T>`.

## `new Store<T>(initialState: T)`

Creates a new store instance with the provided initial state.

```ts
const store = new Store({ count: 0 });
```

### Parameters

- `initialState`: the starting value stored internally by the instance.

## Methods

### `store.getState(): Readonly<T>`

Returns the current state. The return value is typed as `Readonly<T>` to discourage direct mutations.

```ts
const state = store.getState();
console.log(state.count);
```

### `store.getInitialState(): Readonly<T>`

Returns the state provided to the `Store` constructor during initialization.

```ts
const initialState = store.getInitialState();
console.log(initialState.count); // 0
```

### `store.setState(nextState: T | ((prevState: Readonly<T>) => T)): void`

Updates the store state. You can provide either the next value directly or an updater function.

```ts
store.setState({ count: 10 });

store.setState((prev) => ({
  count: prev.count + 1,
}));
```

`setState()` does **not** shallow-merge objects. Whatever you return becomes the next full state value.

If you want to keep existing fields, spread them yourself:

```ts
store.setState((prev) => ({
  ...prev,
  count: prev.count + 1,
}));
```

#### Same-reference bailout

If `Object.is(prevState, nextState)` is `true`, the update is ignored and subscribers are not notified.

Function-valued state is also not a good fit here, because functions passed to `setState()` are always treated as updater functions.

### `store.subscribe(listener: () => void): () => void`

Registers a listener function to be called after every successful state change.

```ts
const unsubscribe = store.subscribe(() => {
  const current = store.getState();
  console.log('State changed to:', current);
});

unsubscribe();
```

- **Execution**: listeners are called synchronously after `setState` completes.
- **Action Name**: the internal signature accepts an optional action name, but the core store does not use it for any behavior.
