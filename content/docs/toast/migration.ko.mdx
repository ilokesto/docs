# 마이그레이션

`@ilokesto/toast`는 `react-hot-toast` 사용자에게 공통 경로에서 익숙하게 느껴지도록 설계되었습니다.

## import 교체

```tsx
// before
import { Toaster, toast } from 'react-hot-toast';

// after
import { Toaster, toast } from '@ilokesto/toast';
```

## 같은 mount 패턴 유지

```tsx
<Toaster />
```

차이는 `Toaster`가 단순 렌더러가 아니라 runtime을 생성하고 등록하는 역할도 한다는 점입니다.

## facade parity

익숙한 facade 호출은 거의 그대로 유지됩니다.

- `toast()`
- `toast.success()`
- `toast.error()`
- `toast.loading()`
- `toast.custom()`
- `toast.promise()`

## 실전 차이점

- `toast.promise()`는 `() => Promise<T>`도 받습니다
- `dismissAll()` / `removeAll()` 같은 별도 메서드는 없고, id 없이 `dismiss()` / `remove()`를 호출합니다
- `useToaster()`는 mounted `Toaster` context 아래에서 실행되어야 합니다
- `transport="top-layer"`는 추가 capability입니다

## `toastOptions` 병합 순서

1. global `toastOptions`
2. per-type `toastOptions.success/error/loading/blank/custom`
3. per-toast call option

## 추천 마이그레이션 순서

1. import를 바꾸고
2. 기존 `toast.*` 호출을 유지하고
3. `<Toaster />`를 마운트하고
4. `toastOptions`를 옮기고
5. advanced custom rendering이 필요할 때만 추가 조정합니다
