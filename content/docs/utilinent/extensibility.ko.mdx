# 확장성

이 페이지는 Utilinent 확장 구조의 개요 페이지입니다. 내장 프리미티브만으로 부족할 때, custom variant와 proxy-backed API 뒤에 어떤 요소가 있는지를 한눈에 설명하는 역할을 합니다.

## `createProxy`

`createProxy(base, renderForTag, category)`는 `Show`, `For` 같은 proxy-backed export를 만드는 함수입니다.

전체 runtime 흐름, 타입 모델, custom component 예제까지 보고 싶다면 [createProxy](./create-proxy)를 읽는 것이 좋습니다.

개요 수준에서 보면 이 함수는 다음을 합칩니다.

- base component
- 생성된 HTML-tag variant
- category, 그리고 그다음 `base` 순서로 해석되는 plugin-backed variant

## `PluginManager`

`PluginManager`는 plugin-backed variant 뒤에 있는 singleton registry입니다.

```ts
import { PluginManager } from '@ilokesto/utilinent';

PluginManager.register({
  show: {
    FancyCard,
  },
  base: {
    MotionButton,
  },
});
```

공개 static 메서드는 다음과 같습니다.

- `register()`
- `get()`
- `getAll()`
- `has()`
- `unregister()`

이 객체는 singleton global state이기 때문에, 한 번 등록된 plugin은 이후의 proxy lookup 전반에 영향을 줍니다.

## export된 타입

루트 패키지는 확장 작업을 위한 타입 헬퍼도 함께 export합니다.

- `UtilinentRegister`
- `BaseTypeHelperFn`
- `ProxyType`

이 타입들은 typed module augmentation이나 proxy-aware API를 만들고 싶을 때 의미가 있습니다. 자세한 타입 설명은 [createProxy](./create-proxy) 페이지에서 다룹니다.

## `useIntersectionObserver`는 어디에 들어가나

이 hook은 사용자도 직접 쓸 수 있지만, `Observer`와 `Slacker`가 이 훅 위에서 동작하기 때문에 확장성 맥락에서도 중요한 위치를 차지합니다.

## 언제 이 페이지를 읽으면 좋은가

이 페이지는 확장 구조를 한눈에 보는 개요입니다. 실제로 “어떻게 하나를 끝까지 만드는가”가 궁금하다면 이어서 [createProxy](./create-proxy)를 읽는 편이 더 좋습니다.
