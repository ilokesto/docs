# Async & Visibility

This part of Utilinent is about output that should depend on timing or visibility.

- `Mount` treats async children as a rendering concern.
- `Observer` turns intersection state into rendering.
- `Slacker` delays expensive loading until a region becomes visible.
- `useIntersectionObserver` is the low-level hook behind that behavior.

## `Mount`

`Mount` accepts either regular children or a function that returns a node or a promise.

```tsx
import { Mount } from '@ilokesto/utilinent';

function AsyncProfile() {
  return (
    <Mount fallback={<p>Loading...</p>}>
      {async () => <section>Loaded profile</section>}
    </Mount>
  );
}
```

The implementation behavior matters here:

- if `children` is already a node, it renders immediately
- if `children` is a function, `fallback` stays visible until the result resolves
- if the function throws or rejects, `fallback` remains and `onError` is called

## `Observer`

`Observer` is the higher-level ‚Äúrender when visible‚Äù component.

```tsx
import { Observer } from '@ilokesto/utilinent';

function LazyImage() {
  return (
    <Observer triggerOnce fallback={<div>Waiting for viewport‚Ä¶</div>}>
      <img src="/hero.png" alt="hero" />
    </Observer>
  );
}
```

It uses `useIntersectionObserver()` internally and applies default `minHeight`, `minWidth`, and `display: block` so the observed element can actually intersect.

## `Slacker`

`Slacker` combines visibility observation with loader execution.

```tsx
import { Slacker } from '@ilokesto/utilinent';

function LazyPanel() {
  return (
    <Slacker
      loader={async () => ({ title: 'Loaded' })}
      loadingFallback={<p>Loading‚Ä¶</p>}
      errorFallback={({ retry }) => <button onClick={retry}>Retry</button>}
    >
      {(data) => <section>{data.title}</section>}
    </Slacker>
  );
}
```

It keeps `null` as the ‚Äúnot loaded yet‚Äù sentinel and supports retry behavior through `maxRetries` and `retryDelay`.

## `useIntersectionObserver`

Use the hook directly when you want intersection state without the opinionated rendering wrappers.

```tsx
import { useIntersectionObserver } from '@ilokesto/utilinent';

function Sentinel() {
  const { ref, isIntersecting, entry } = useIntersectionObserver({
    threshold: 0.5,
    rootMargin: '50px',
  });

  return <div ref={ref}>{isIntersecting ? 'Visible' : 'Hidden'}</div>;
}
```

Notable behaviors:

- the first callback is skipped
- `freezeOnceVisible` stops future observation after the first visible state
- state resets when the observed node is cleared

If the next problem is not visibility but prop/ref composition, continue to [Composition](./composition).
