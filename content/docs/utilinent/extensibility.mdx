# Extensibility

This page is for the part of Utilinent that regular consumers may never touch directly: the machinery behind proxy-backed variants and plugin-based extension.

## `createProxy`

`createProxy(base, renderForTag, category)` is the function that turns a base primitive into a proxy-backed export such as `Show` or `For`.

It builds a target from:

- the base component,
- generated HTML-tag variants,
- plugin-backed variants resolved at access time.

The resolution order is important:

1. existing properties on the target
2. plugins registered for the current category
3. plugins registered for `base`

That means built-ins win before plugin lookups happen.

## `PluginManager`

`PluginManager` is the singleton registry behind plugin-backed variants.

```ts
import { PluginManager } from '@ilokesto/utilinent';

PluginManager.register({
  show: {
    FancyCard,
  },
  base: {
    MotionButton,
  },
});
```

Public static methods include:

- `register()`
- `get()`
- `getAll()`
- `has()`
- `unregister()`

Because it is singleton global state, registrations affect later proxy lookups across the app.

## Exported types

The root package also exports type helpers for extension work:

- `UtilinentRegister`
- `BaseTypeHelperFn`
- `ProxyType`

These are relevant when you want typed module augmentation or your own proxy-aware APIs.

## Where `useIntersectionObserver` fits

The hook is user-facing, but it also belongs to the extension story because `Observer` and `Slacker` build on top of it rather than duplicating observer logic.

## When to read this page

Read this page if you want to extend Utilinent itself. If you only want to consume the existing primitives, the earlier sections are the better starting point.
