# 디바운스 (Debounce)

`debounce` 미들웨어는 텍스트 입력이나 브라우저 크기 조정과 같은 빈번한 상태 업데이트를 큐에 모아 두었다가 일정 시간 동안 추가 업데이트가 없을 때만 최종적으로 반영합니다.

```ts
import { debounce } from '@ilokesto/state/middleware';

const store = debounce(200)({ text: '' });
```

## 동작 원리

1. **큐잉 (Queueing)**: 설정된 대기 시간 동안 업데이트 시도를 수집합니다.
2. **리플레이 (Replaying)**: 큐에 쌓인 함수형 업데이트는 수집된 순서대로 현재 누적된 상태에 대해 다시 계산됩니다.
3. **플러싱 (Flushing)**: 대기 시간이 만료되고 새로운 업데이트가 없을 때, 최종적으로 계산된 상태가 스토어에 반영됩니다.

## 호출 형태

- 커링 형태: `debounce(wait)(store)`
- 직접 호출 형태: `debounce(store, wait)`

대기 시간을 생략하면 현재 구현은 `300`ms를 기본값으로 사용합니다.

## 주의 사항

- **지연 (Latency)**: `debounce`는 의도적으로 쓰기 작업을 지연시킵니다. 즉각적인 동기적 상태 접근이 필요한 경우(예: 즉각적인 피드백이 필요한 제어 컴포넌트)에는 사용하지 마십시오.
- **순차 처리**: 함수형 업데이트를 순차적으로 리플레이하여 디바운싱 중에도 일관된 상태 전이를 보장합니다.
