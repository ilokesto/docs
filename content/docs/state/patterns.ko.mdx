# 패턴 (Patterns)

`@ilokesto/state`는 유연한 아키텍처 패턴을 지원합니다. 여러분의 애플리케이션의 필요에 맞는 패턴을 선택해 보세요.

## 별도의 액션 정의

상태 정의와 업데이트 로직을 분리하여 테스트 가능성을 높이고 스토어 설정을 간단하게 유지할 수 있습니다.

```ts
// stores/counter.ts
export const useCounter = create({ count: 0 });

// actions/counter.ts
import { useCounter } from '../stores/counter';

export const increment = () => {
  useCounter.writeOnly()(state => ({ count: state.count + 1 }));
};
```

## 파생 상태 (Memoization)

파생된 상태를 위해 간단한 셀렉터를 사용하십시오. 비용이 많이 드는 계산은 React 외부에서 메모이제이션(memoize) 하세요.

```ts
const useStore = create({ items: [] });

// 간단한 파생 상태
const useTotalItems = () => useStore(state => state.items.length);

// 비용이 많이 드는 계산 (메모이제이션)
const selectExpensiveData = (state) => {
  // 무거운 계산 작업 수행...
};
const useExpensiveData = () => useStore(selectExpensiveData);
```

## 모듈형 스토어

하나의 거대한 스토어 대신 여러 개의 작고 집중된 스토어를 사용하십시오. 이를 통해 애플리케이션의 모듈성을 유지하고 상태 변화를 더 쉽게 관리할 수 있습니다.

```ts
// auth.ts
export const useAuth = create({ user: null });

// products.ts
export const useProducts = create({ list: [] });
```

## 효과(Effects)와 결합

액션에서 직접 사이드 이펙트를 발생시키거나 `useEffect`를 사용하여 컴포넌트 내에서 처리할 수 있습니다.

```ts
// 사이드 이펙트가 있는 액션
export const fetchUser = async (id: string) => {
  const user = await api.getUser(id);
  useAuth.writeOnly()({ user });
};
```
