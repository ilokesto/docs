# 빠른 시작

이 페이지는 최소한의 맥락으로 동작하는 toast stack 하나를 화면에 올리는 데 집중합니다.

```package-install
@ilokesto/toast
```

## 1. `Toaster` 마운트하기

`toast.*` 호출은 runtime이 등록되어 있어야만 동작합니다. 실제로는 `<Toaster />`를 마운트해야 한다는 뜻입니다.

```tsx
import { Toaster } from '@ilokesto/toast';

function App() {
  return <Toaster />;
}
```

## 2. facade 호출하기

```tsx
import { toast } from '@ilokesto/toast';

toast.success('저장되었습니다');
```

이것만으로도 기본 runtime에 토스트가 표시됩니다.

## 3. 다른 내장 variant도 사용하기

```tsx
toast('Hello');
toast.error('문제가 발생했습니다');
toast.loading('저장 중...');
```

`toast.loading()`은 다른 동작이 갱신하거나 dismiss할 때까지 무한 duration을 사용합니다.

## 4. promise toast도 익숙한 경로에 가깝다

```tsx
toast.promise(fetch('/api/save'), {
  loading: '저장 중…',
  success: '저장 완료',
  error: '실패',
});
```

loading -> success/error 전환에서 같은 toast id가 재사용됩니다.
