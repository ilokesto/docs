# @ilokesto/utilinent

`@ilokesto/utilinent`는 React에서 반복해서 나타나는 렌더링 로직을 다루기 위한 유틸리티 라이브러리입니다. 조건 분기, 컬렉션 렌더링, 비동기 마운팅, 가시성 기반 로딩, 슬롯 컴포지션, 프록시 기반 변형처럼 JSX에서 자주 지저분해지는 패턴에 이름을 붙여 줍니다.

이 패키지를 이해할 때 중요한 점은 이것이 state 라이브러리도 아니고 styling 라이브러리도 아니라는 점입니다. 핵심은 **렌더링 로직**입니다. 자주 반복되는 JSX 제어 흐름을 더 명시적인 형태로 바꾸는 역할을 합니다.

## 이 패키지에 들어 있는 것

- `Show`, `Switch`, `Match`, `For`, `Repeat`, `Mount`, `OptionalWrapper` 같은 **렌더링 프리미티브**
- `Observer`, `Slacker`, `useIntersectionObserver` 같은 **가시성 헬퍼**
- `Slot`, `Slottable` 같은 **컴포지션 헬퍼**
- `createProxy`, `PluginManager`, helper type exports 같은 **확장 메커니즘**

## 루트 import 모델

`package.json` 기준으로 이 패키지는 루트 엔트리포인트 하나만 export합니다. 그래서 학습도 root import부터 시작하는 것이 가장 자연스럽습니다.

```package-install
@ilokesto/utilinent
```

```tsx
import { For, Show, Slot } from '@ilokesto/utilinent';
```

## 왜 이런 헬퍼가 필요한가

핵심은 추상적인 “유틸리티 컴포넌트”가 아니라, JSX가 복잡한 렌더링 규칙을 만나면 빠르게 소음이 커진다는 데 있습니다.

- 삼항 연산자와 `&&` 체인은 의도를 흐리고,
- `map()`과 빈 상태 처리는 곳곳에 반복되며,
- 가시성/지연 로딩은 쉽게 잘못 연결되고,
- as-child 컴포지션은 slot primitive 없이는 자주 어색해집니다.

Utilinent는 React의 렌더링 모델 자체를 바꾸지 않습니다. 다만 이런 패턴을 더 명시적인 형태로 드러내 줍니다.
