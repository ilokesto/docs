# @ilokesto/store

`@ilokesto/store` is the smallest package in the ilokesto state family. It gives you one class — `Store<T>` — with state storage, replacement, and subscriptions.

It does not include React hooks, selectors, middleware, persistence, or devtools integration. That is exactly why it is useful: it is the low-level primitive that higher-level packages can build on.

## Installation

```bash
pnpm add @ilokesto/store
```

## Mental model

The mental model is simple:

1. create a store with an initial state
2. read the current state with `getState()`
3. replace the state with `setState(...)`
4. subscribe to changes with `subscribe(...)`

### Key characteristics

- **Vanilla JS**: zero dependencies on React or any other framework.
- **Synchronous**: state updates and listener notifications happen synchronously.
- **Reference equality**: updates are ignored if the new state is strictly equal (`Object.is`) to the current state.
- **Type-safe**: TypeScript types describe the stored shape and updater contract.

## Basic example

```ts
import { Store } from '@ilokesto/store';

interface CounterState {
  count: number;
}

const counterStore = new Store<CounterState>({ count: 0 });

const unsubscribe = counterStore.subscribe(() => {
  console.log('State updated:', counterStore.getState());
});

counterStore.setState({ count: 1 });
counterStore.setState((prev) => ({ count: prev.count + 1 }));

unsubscribe();
```

## When to use it

Use `@ilokesto/store` when you need a raw state container that does not care about the UI layer.

It is a good fit for:

- non-React or framework-agnostic state
- custom bindings for another framework
- small shared stores where full middleware or React integration would be overkill

If you want hook-based access in React, use `@ilokesto/state` on top of it.
