# 멘탈 모델 (Mental Model)

`@ilokesto/toast`의 아키텍처를 이해해 봅시다.

## 1. `toasterId`를 통한 런타임 레지스트리

토스트는 하나의 전역 스토어에서 관리되지 않습니다. 대신, 각 `Toaster` 컴포넌트가 고유한 **런타임**을 레지스트리에 등록합니다. `toast.success()`를 호출하면 라이브러리는 `toasterId` (기본값은 `"default"`)를 기반으로 올바른 런타임을 찾습니다.

이를 통해 애플리케이션 내에 독립적인 다중 토스트 영역을 가질 수 있습니다.

## 2. 퍼사드/런타임 분리 (Facade/Runtime Split)

가져오는 `toast` 객체는 **퍼사드(facade)**입니다. 스스로 어떠한 상태도 가지고 있지 않으며, 단순히 사용자의 명령을 적절한 런타임으로 전달합니다. 런타임은 특정 `Toaster`에 대한 토스트 상태를 관리하는 역할을 담당합니다.

## 3. 스토어 + 오버레이 존재 유무 분리

내부적으로 각 런타임은 두 가지 다른 스토어를 사용합니다.
- **토스트 스토어 (Toast Store)**: 각 토스트의 내용, 타이머, 특정 옵션을 관리합니다.
- **오버레이 스토어 (Overlay Store)**: `@ilokesto/overlay`를 사용하여 토스트 항목의 존재(마운트/언마운트)와 생명주기를 관리합니다.

## 4. 옵션 병합 순서 (Options Merge Order)

토스트 옵션은 다음 순서대로 병합됩니다.
1. **전역 기본값 (Global Defaults)**: 라이브러리에 내장된 기본값입니다.
2. **Toaster Props**: `<Toaster toastOptions={...} />` 컴포넌트에 전달된 옵션입니다.
3. **유형별 기본값 (Type Defaults)**: 특정 토스트 유형(예: `toastOptions.success`)에 대한 옵션입니다.
4. **개별 토스트 옵션 (Per-toast Options)**: `toast()` 호출 시 직접 전달된 옵션입니다.

## 5. Dismiss vs Remove

- **Dismiss**: "소프트(soft)" 닫기입니다. 토스트를 `closing` 상태로 표시하고 타이머를 중지하며 종료 애니메이션을 트리거합니다. 토스트는 완전히 제거되기 전에 짧은 기간(기본값 1000ms) 동안 스토어에 남아 있습니다.
- **Remove**: "하드(hard)" 삭제입니다. 토스트가 스토어와 UI에서 즉시 제거됩니다.
