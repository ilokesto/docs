# 정신 모델 (Mental Model)

`@ilokesto/state`는 간단하지만 강력한 원칙 위에 구축되었습니다: 상태는 외부에 존재해야 하지만, 그 구독은 React의 렌더링 생명주기와 깊게 통합되어야 한다는 것입니다.

## 스토어-훅의 통합

애플리케이션을 `Provider`로 감싸야 하는 많은 상태 관리 라이브러리와 달리, `@ilokesto/state`는 **스토어-훅의 통합** 방식을 사용합니다.

1. **스토어 (Store)**: 데이터에 대한 간단한 컨테이너로, React 외부에 존재합니다. 리스너(listeners)와 동기적 업데이트를 지원합니다.
2. **훅 (Hook)**: `create`에 의해 반환되는 특수한 React 훅으로, 스토어의 특정 부분을 컴포넌트에 연결합니다.

## 셀렉터 (Selectors)가 핵심입니다

최적의 성능을 보장하기 위해 `@ilokesto/state`는 **셀렉터** 사용을 권장합니다. 전체 스토어를 구독하는 대신, 특정 조각(slice)만 구독하게 됩니다.

- **선택적 렌더링**: 컴포넌트는 자신이 선택한 특정 데이터가 변경될 때만 리렌더링됩니다.
- **깊은 비교 (Deep Comparison)**: 기본적으로 라이브러리는 엄격한 일치(`===`)를 사용하여 선택된 데이터를 비교합니다. 더 복잡한 객체의 경우 내부적으로 깊은 비교를 수행합니다.

## 데이터 흐름

- **읽기**: 컴포넌트는 셀렉터를 사용하여 훅을 통해 상태를 읽습니다. 이는 내부적으로 `useSyncExternalStore`를 사용하여 구독을 생성합니다.
- **쓰기**: 상태는 함수 기반의 `setState`(React의 `useState`와 유사)를 사용하여 업데이트됩니다. 쓰기는 컴포넌트 내부나 외부 어디에서든 발생할 수 있습니다.
- **미들웨어**: 스토어 수준에서 데이터 흐름을 가로채어 로깅이나 영속화와 같은 부가 효과(side effects)를 추가합니다.

## 보일러플레이트 제거

기본적으로 "Reducer"(원하는 경우 사용 가능), "Action", "Dispatcher"와 같은 복잡한 개념이 없습니다. 단순히 상태를 정의하고 업데이트하기만 하면 됩니다. 따라서 대규모 프레임워크의 오버헤드 없이 강력한 상태 관리가 필요한 소규모 및 중규모 애플리케이션에 적합합니다.
