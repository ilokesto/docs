# 멘탈 모델

`@ilokesto/store`는 하나의 동기적 값 컨테이너라고 생각하면 가장 이해하기 쉽습니다.

이 패키지는 세 가지만 합니다.

- 값을 보관하고,
- 그 값을 교체하고,
- 값이 바뀌면 리스너에게 알립니다.

그 외의 것은 의도적으로 범위 밖에 둡니다.

## 하나의 atom이지, 프레임워크 계층이 아니다

`Store`는 React, Vue, reducer, persistence, selector를 알지 못합니다. 그런 것은 더 높은 계층이 맡아야 합니다.

그래서 API도 아주 작습니다.

- `getState()`
- `getInitialState()`
- `setState()`
- `subscribe()`

## 읽기는 단순하다

여기에는 파생 읽기 시스템이 없습니다. 읽는다는 것은 단순히 현재 값을 준다는 뜻입니다.

더 좁은 읽기가 필요하다면, 그건 상위 바인딩이나 직접 만든 adapter의 책임입니다.

## 쓰기는 값을 교체한다

가장 중요한 멘탈 모델은 store write가 상태 값을 교체한다는 점입니다. 객체를 알아서 merge해 주지 않습니다.

그래서 객체 업데이트는 보통 이렇게 씁니다.

```ts
store.setState((prev) => ({ ...prev, count: prev.count + 1 }));
```

## 구독은 동기적이다

`Store`는 업데이트를 프레임워크 스케줄러로 넘겨 큐잉하지 않습니다. 성공한 쓰기는 즉시 상태를 바꾸고, 그 다음 리스너를 알립니다.

이 점 덕분에 다른 계층이 그 위에 쌓이기 좋지만, 동시에 mutation이나 listener 내부 작업을 더 신중하게 다뤄야 합니다.

## 왜 이 패키지는 작게 남아 있나

`@ilokesto/store`의 목적은 상태 관리 전체를 혼자 담당하는 것이 아닙니다. 다른 패키지가 그 위에 안정적으로 쌓일 수 있는 기반 계층이 되는 것이 목적입니다.

정확한 동작 규칙을 보고 싶다면 [시맨틱](./semantics)을, 실전 패턴을 보고 싶다면 [패턴](./patterns)을 이어서 읽으면 됩니다.
