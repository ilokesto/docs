# 영속화 (Persist)

`persist` 미들웨어는 브라우저 세션 전반에 걸쳐 스토어 상태를 자동으로 저장하고 복원합니다.

```ts
import { persist } from '@ilokesto/state/middleware';

const store = persist({ local: 'counter-storage' })({ count: 0 });
```

## 옵션

- `local`: (string) `localStorage`에 저장할 키 이름입니다.
- `session`: (string) `sessionStorage`에 저장할 키 이름입니다.
- `cookie`: (string) 쿠키 저장소에 저장할 키 이름입니다.
- `migrate`: (함수 배열) 선택적인 마이그레이션 체인입니다. 현재 타입 모델에서는 `local`과 `cookie` 경로에서만 마이그레이션을 지원하고 `session`은 지원하지 않습니다.

## 구현 상세 내용

- **초기화**: 스토어 설정 단계에서 구성된 저장소로부터 영속화된 상태를 읽어옵니다.
- **저장소 형태**: 데이터는 `{ state, version }` 객체 형태로 저장됩니다.
- **쓰기 최적화**: 메모리 기반의 쓰기 캐시(write cache)를 사용하여 중복된 쓰기 작업을 건너뛰고 저장소 접근을 최소화합니다.
- **마이그레이션**: 현재 버전에서는 `local` 및 `cookie` 저장소 경로에 대해서만 마이그레이션을 적용합니다.
- **업데이트 사이클**: 스토어의 상태 참조가 실제로 변경되었을 때만 저장소에 다시 씁니다.
- **버전 계산 방식**: 저장되는 `version` 값은 별도의 공개 옵션이 아니라 마이그레이션 체인의 길이에서 계산됩니다.

## 권장 사항

- 시간이 지나면서 스토어의 상태 구조가 변경될 것으로 예상되는 경우 `migrate`를 사용하십시오.
- 다른 애플리케이션과의 충돌을 피하기 위해 저장소 키 이름을 명시적으로 지정하십시오.
- **영속화 순서**: 다른 미들웨어에 의해 처리된 최종 상태가 저장되도록 미들웨어 체인의 끝부분에 `persist()`를 배치하십시오.
