# SSR 및 초기 상태 (SSR and Initial State)

`@ilokesto/state`는 `useSyncExternalStore`를 사용하여 서버 사이드 렌더링(SSR)과 호환됩니다.

## 하이드레이션 (Hydration)

상태를 올바르게 하이드레이션하려면 클라이언트 측 스토어가 서버에서 보낸 데이터로 초기화되도록 해야 합니다.

```ts
// 서버 사이드
const initialData = { count: 42 };

// 클라이언트 사이드
const useCounter = create(initialData);
```

## `getServerSnapshot`

내부적으로 `@ilokesto/state`는 React의 `useSyncExternalStore`에 대한 `getServerSnapshot` 요구 사항을 충족하기 위해 `store.getInitialState()`를 사용합니다. 이를 통해 서버에서 렌더링된 HTML이 클라이언트의 초기 상태와 일치하도록 보장하고, 하이드레이션 불일치(mismatch)를 방지합니다.

## 권장 사항

1. **서버에서의 전역 상태 지양**: 서버에서 실행될 때, 사용자 간의 데이터 유출을 방지하기 위해 각 요청이 자체 스토어 인스턴스를 갖도록 하십시오.
2. **버전 동기화**: 애플리케이션에서 영속화(persistence)를 사용하는 경우, 서버 버전의 상태가 클라이언트가 기대하는 것과 충돌하지 않도록 하십시오.
3. **지연 초기화 (Lazy Initialization)**: 경우에 따라 데이터 어트리뷰트(data-attribute)나 전역 `window.__INITIAL_STATE__` 변수로부터 상태를 초기화하고 싶을 수 있습니다.
