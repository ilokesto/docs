# Toaster

`Toaster`는 단순한 시각적 컨테이너가 아닙니다. runtime 하나를 만들고, `toasterId`로 등록하고, view option을 설정하고, 현재 visible toast stack을 렌더링하는 component입니다.

## 핵심 props

- `toasterId?`
- `position?`
- `transport?`
- `limit?`
- `reverseOrder?`
- `gutter?`
- `containerStyle?`
- `containerClassName?`
- `toastOptions?`
- `children?`

## runtime 등록

`Toaster`는 `useToasterRuntime(toasterId)`로 runtime을 얻고, 그 runtime은 전역 registry에 등록됩니다. 그래서 이후 facade 호출이 runtime을 찾을 수 있습니다.

주어진 `toasterId`에 해당하는 runtime이 마운트되어 있지 않으면 facade는 오류를 던집니다.

## position, limit, order

- `position`은 이 렌더링 스택에 어떤 runtime item이 보일지 필터링하고
- `limit`은 visible subset만 제한하며 raw store 전체를 지우지 않고
- `reverseOrder`는 렌더링 순서만 바꾸고
- `gutter`는 측정된 row 간 offset 계산에 영향을 줍니다

## `transport`

- `inline`은 일반 React 트리에서 렌더링하고
- `top-layer`는 지원되는 브라우저에서 manual popover를 쓰고, 아니면 inline으로 fallback합니다

## `toastOptions`

`toastOptions`는 이 runtime view layer의 global/per-type 기본값을 제공합니다.

per-toast call option은 그 위에 마지막으로 병합됩니다.

## custom row rendering

`children`이 주어지면 `Toaster`는 기본 `ToastBar` 대신 그 render function을 사용합니다.

```tsx
<Toaster>
  {(toast, { dismiss }) => (
    <button onClick={dismiss}>{toast.message}</button>
  )}
</Toaster>
```

## hover pause

기본 컨테이너는 mouse enter 시 runtime timer를 멈추고, mouse leave 시 다시 시작합니다.
