`@ilokesto/store`는 작은 코어 역할을 하도록 설계되었습니다. 이 페이지는 그 코어 위에 놓이는 패키지와 어댑터 계층을 소개하는 허브입니다.

### 이 페이지를 읽는 방법

이 페이지는 의도적으로 짧게 유지합니다. 각 통합을 여기서 모두 설명하지 않고, 해당 사용 사례를 실제로 책임지는 계층으로 안내하는 역할만 합니다.

### 현재 패키지

#### `@ilokesto/state`

`@ilokesto/store` 위에 구축된 상위 수준의 상태 패키지입니다.

- 바닐라 store보다 더 풍부한 상태 API가 필요할 때 사용합니다.
- React용 `create()` 흐름, selector, reducer 모드, middleware 유틸리티는 해당 패키지 문서를 참고하십시오.

### 향후 통합

- **폼 라이브러리**는 값을, 오류를, 제출 생명주기를 관리하는 작은 상태 엔진으로 스토어를 사용할 수 있습니다.
- **오버레이나 모달 시스템**은 열림 상태와 payload를 조정하는 데 사용할 수 있습니다.
- **프레임워크 어댑터**는 `getState`, `setState`, `subscribe`를 각 프레임워크의 반응형 API로 변환할 수 있습니다.

### 직접 무언가를 만들고 있다면

새 패키지를 `@ilokesto/store` 위에 만들고 있다면 [스토어 기반 구축하기](./building-on-store)부터 읽는 것이 좋습니다. 이 페이지는 어댑터와 상위 라이브러리가 의존해야 하는 안정적인 계약을 설명합니다.

이 integrations 페이지는 디렉터리 역할을 하는 곳이지, 모든 소비자 패키지의 구현 가이드를 대신하는 곳은 아닙니다.
