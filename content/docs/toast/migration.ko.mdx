# 마이그레이션 (Migration)

`react-hot-toast`에서 오시는 분들을 위해, `@ilokesto/toast`는 더 강력한 기능과 익숙한 API를 제공합니다.

## 주요 차이점

| 기능 | `react-hot-toast` | `@ilokesto/toast` |
| --- | --- | --- |
| **API** | 전역 싱글톤 `toast` | 퍼사드 `toast` + 레지스트리 |
| **범위 지정** | 전역 전용 | `toasterId`를 통한 프로바이더 범위 |
| **런타임** | 내부 상태 | `@ilokesto/store` + `@ilokesto/overlay` 기반 |
| **탑 레이어** | 미지원 | 선택적 `transport="top-layer"` 지원 |
| **헤드리스** | `useToaster` | `useToaster` + `useToastItems` |

## API 매핑 (Mapping)

대부분의 메서드는 1:1로 호환됩니다.

```ts
// react-hot-toast
import toast from 'react-hot-toast';
toast.success('완료!');

// @ilokesto/toast
import { toast } from '@ilokesto/toast';
toast.success('완료!');
```

## `<Toaster />` 이동하기

`react-hot-toast`에서 `<Toaster />`는 단순한 렌더러입니다. `@ilokesto/toast`에서는 런타임을 생성하고 등록하는 역할도 담당합니다.

애플리케이션 최상단(또는 토스트를 표시하려는 영역)에 `<Toaster />`를 마운트해야 합니다.

```tsx
// react-hot-toast
import { Toaster } from 'react-hot-toast';
<Toaster position="top-center" />

// @ilokesto/toast
import { Toaster } from '@ilokesto/toast';
<Toaster position="top-center" />
```

## 커스텀 렌더링 (Custom Rendering)

`react-hot-toast`는 `<Toaster />`에서 `children` 렌더 프롭을 사용하는 반면, `@ilokesto/toast`는 완전한 재정의를 위해 `toast.custom()`을 사용하거나 레이아웃 수정을 위해 `ToastBar` 컴포넌트의 `children` 프롭을 사용하는 것을 선호합니다.

더 자세한 내용은 저장소의 [MIGRATION_FROM_REACT_HOT_TOAST.md](https://github.com/ilokesto/ilokesto/blob/main/packages/toast/MIGRATION_FROM_REACT_HOT_TOAST.md)를 참조하세요.
