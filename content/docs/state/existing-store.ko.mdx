# 기존 스토어 사용

`@ilokesto/state`는 꼭 plain initial state에서 시작할 필요가 없습니다. 이미 vanilla `Store` 인스턴스를 갖고 있다면, 그 스토어를 다시 만들지 않고 React hook 계층만 얹을 수 있습니다.

## 기본 모양

`create` 함수는 초기 상태 또는 기존 `Store` 인스턴스를 모두 인자로 받을 수 있습니다.

```ts
import { Store } from '@ilokesto/store';
import { create } from '@ilokesto/state';

// 1. vanilla store 생성
const myStore = new Store({ count: 0 });

// 2. React hook 계층 연결
const useCounter = create(myStore);
```

## 왜 래핑하나요?

- **스토어를 이미 다른 곳에서 만들고 있을 때**
- **React와 non-React 코드가 하나의 source of truth를 공유해야 할 때**
- **React 전에 미들웨어 구성을 끝내고 싶을 때**
- **`@ilokesto/state`를 store 생성기가 아니라 얇은 binding 계층으로 유지하고 싶을 때**

## 바뀌지 않는 것

기존 `Store`를 전달하면 `create`는 두 번째 사본을 만들지 않고 그 인스턴스를 그대로 사용합니다.

- 기존 미들웨어는 계속 동작하고,
- 기존 상태도 유지되며,
- SSR snapshot에도 여전히 `getInitialState()`가 중요합니다.

대신 수명과 소유권은 이제 React 계층이 아니라 원래 store를 만든 쪽에 있습니다.
