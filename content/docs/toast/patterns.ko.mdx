# 패턴

## promise toast

가장 흔한 상위 패턴은 `toast.promise()`입니다.

```ts
toast.promise(fetchData(), {
  loading: '데이터를 가져오는 중...',
  success: (data) => `${data.name}를 불러왔습니다`,
  error: '데이터를 가져오지 못했습니다.',
});
```

runtime은 loading -> success/error 동안 같은 toast id를 재사용합니다.

## 개별 토스트 스타일링

```ts
toast.success('삭제됨!', {
  style: { background: 'red', color: 'white' },
  iconTheme: { primary: 'white', secondary: 'red' },
});
```

하나의 토스트만 다른 모양이 필요할 때 적당한 수준입니다.

## 여러 toaster 함께 쓰기

```tsx
<>
  <Toaster toasterId="app" position="top-right" />
  <Toaster toasterId="editor" position="bottom-center" />
</>

toast.success('저장됨', { toasterId: 'editor' });
```

각 `toasterId`는 독립적인 runtime을 가집니다.

## 모두 dismiss vs 모두 remove

```ts
toast.dismiss();
toast.remove();
```

id를 생략하면 해당 runtime의 active toast 전체에 적용됩니다.

## `Toaster`를 통한 custom row

```tsx
<Toaster>
  {(toast, { dismiss }) => (
    <button onClick={dismiss}>{toast.message}</button>
  )}
</Toaster>
```

기본 presentation보다 row 제어가 더 중요할 때 이 경로가 맞습니다.
