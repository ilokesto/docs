# @ilokesto/state

`@ilokesto/state`는 `@ilokesto/store` 위에 놓이는 React 계층입니다. 새로운 상태 컨테이너를 하나 더 만드는 대신, 외부 스토어 모델을 `useSyncExternalStore` 기반의 React 훅 API로 연결하는 역할을 합니다.

이 관점을 먼저 잡아두면 패키지의 나머지 설계도 자연스럽게 이해할 수 있습니다. 스토어는 React 바깥에 남고, 읽기와 구독만 훅으로 표현됩니다.

## 이 패키지가 제공하는 것

- 상태나 vanilla store를 React 훅으로 바꾸는 `create()` 진입점
- plain state mode와 reducer mode
- 읽기 범위를 좁혀 주는 selector 기반 구독
- 컴포넌트 밖 액션을 위한 `writeOnly()` 같은 접근자
- 로깅, 영속화, 검증, 디버깅을 위한 store-level 미들웨어

## 의도적으로 제공하지 않는 것

- Provider 중심 구조
- Zustand 스타일 initializer function
- selector용 별도 custom equality API
- 훅 아래에 실제 store가 있다는 사실을 감추는 추상화

## 설치

```package-install
@ilokesto/state
```

## 훑어보기

```tsx
import { create } from '@ilokesto/state';

const useCounter = create({ count: 0 });

function Counter() {
  const [count, setState] = useCounter((state) => state.count);

  return (
    <button onClick={() => setState((prev) => ({ ...prev, count: prev.count + 1 }))}>
      {count}
    </button>
  );
}
```

이 짧은 예제 안에도 패키지의 핵심 모양이 다 들어 있습니다.

- `create()`가 외부 상태를 훅으로 바꾸고,
- selector가 컴포넌트가 읽을 범위를 정하고,
- writer는 여전히 전체 store state를 갱신합니다.
