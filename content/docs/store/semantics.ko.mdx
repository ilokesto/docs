# 상태 시맨틱

이 Store는 상태를 **불변의 스냅샷**으로 취급합니다. 상태를 직접 수정하지 말고 항상 `setState()`를 통해 업데이트하세요.

## 주의사항 및 FAQ

- **동일 참조 최적화**: 동일한 참조를 제공하면 변경으로 간주되지 않아 구독자에게 알림이 가지 않습니다.
- **함수형 상태**: `setState()`에 전달된 함수는 항상 업데이터로 취급됩니다. 현재 API는 상태 값 자체가 함수인 패턴에는 적합하지 않습니다.
- **런타임 불변성**: 스토어는 런타임에서 깊은(deep) 불변성을 강제하지 않습니다. `Readonly<T>`는 TypeScript 수준의 구성입니다.

## 미들웨어 동작

현재 버전의 `@ilokesto/store`는 스토어 인스턴스에 직접 미들웨어 체인을 등록할 수 있습니다.

- 미들웨어는 `Array`에 저장되므로 등록 순서가 유지됩니다.
- 같은 미들웨어를 여러 번 등록할 수도 있습니다.
- `pushMiddleware()`는 뒤에 추가하고, `unshiftMiddleware()`는 앞에 추가합니다.

`setState()`는 내부적으로 `reduceRight()`로 실행 체인을 만들기 때문에, 배열 앞쪽에 있는 미들웨어가 먼저 업데이트를 보게 됩니다.
