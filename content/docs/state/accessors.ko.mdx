# 접근자

접근자는 `create`가 반환하는 훅에 붙는 메서드입니다. 다만 둘의 성격은 다릅니다. `writeOnly()`는 안정적인 imperative writer이지만, `readOnly()`는 현재 일반 selector 읽기와 같은 hook 경로를 통해 동작합니다.

## `writeOnly()`

`writeOnly()`는 스토어의 `setState` 함수를 반환합니다. 이는 React 외부에서 정의된 액션이나 부가 효과(side effects)를 관리하는 가장 권장되는 방법입니다.

```ts
const useCounter = create({ count: 0 });

// React 외부에서 액션 정의
export const increment = () => {
  const setState = useCounter.writeOnly();
  setState(state => ({ count: state.count + 1 }));
};
```

## `readOnly()`

`readOnly()`는 반환된 훅에 존재하지만, 현재 구현은 일반 selector 읽기와 같은 hook 경로를 다시 사용합니다.

즉 이것을 임의의 imperative getter처럼 생각하면 안 되고, React 안에서 쓰는 편의 기능 정도로 보는 편이 맞습니다.

```tsx
function CounterSummary() {
  const count = useCounter.readOnly((state) => state.count);

  return <span>{count}</span>;
}
```

React 바깥에서 imperative read가 필요하다면 `@ilokesto/store`의 vanilla `Store` API를 직접 쓰고, 그 스토어를 `create(store)`에 전달하는 편이 더 안전합니다.

## 액션에서의 활용

접근자는 `writeOnly()`를 통해 독립적인 액션 파일을 정의할 때 특히 유용합니다:

```ts
// actions/counter.ts
import { useCounter } from '../stores/counter';

export const reset = () => {
  useCounter.writeOnly()({ count: 0 });
};
```
