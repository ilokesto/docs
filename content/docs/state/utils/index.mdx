# Utilities

The `@ilokesto/state/utils` entry point currently exposes two helpers: `pipe` and `adaptor`.

These helpers do different jobs:

- [`pipe`](./pipe) composes middleware around plain initial state and returns a `Store`.
- [`adaptor`](./adaptor) wraps Immer's `produce()` so you can write draft-style object updates.

## When to reach for utilities

Use `utils` when you want to improve store setup ergonomics or object update ergonomics without changing the public `create()` API.

## Related APIs

- [`create`](../create): Main entry point for React integration.
- [`middleware`](../middleware): Built-in middleware for common tasks.

```ts
import { create } from '@ilokesto/state';
import { pipe } from '@ilokesto/state/utils';
import { logger, devtools } from '@ilokesto/state/middleware';

const store = pipe(
  { count: 0 },
  logger(),
  devtools('MyStore')
);

const useStore = create(store);
```

Using `pipe` is the recommended way to set up complex stores with multiple middleware.

## `adaptor`

`adaptor` is a thin wrapper around Immer's `produce`. It lets you write a draft-style state recipe and get back a pure updater function.

```ts
import { adaptor } from '@ilokesto/state/utils';

type State = { user: { profile: { name: string } } };

const renameProfile = adaptor<State>((draft) => {
  draft.user.profile.name = 'New Name';
});

useStore.writeOnly()(renameProfile);
```

### Installation

`adaptor` requires `immer` to be installed in your project.

```package-install
immer
```
